
# AI 개발 오케스트레이션 워크플로우 요구사항
* 기존 스펙 문서와 차이를 파악하고 페이즈별로 요구사항을 반영해라.

## 큰그림
1. 사용자는 칸반보드에 접근하여 프로젝트를 선택하고 테스크를 만들고 검수하는 것만으로 모든 작업이 끝난다.
2. 테스크 생성 - ready 상태
   1. 클러드에서 해당 테스크를 읽어 sub agent, skill, mcp, plugin 등 만들어둔 기능들을 활용하여 작업을 진행한다.(In Progress 상태)
3. In Progress 상태
   1. 클러드는 작업 진행간 변경사항은 CHANGELOG로 남기고, 랜딩페이지에 적층하며 표시한다.
   2. 클러드는 작업 완료 시 취약점 검토 및 디버깅 진행 후 해당 변경사항에 대한 코드리뷰를 진행한다.
   3. 모든 단계가 완료되면 review 상태로 변경 후 사용자 피드백을 받는다.
4. review 상태
   1. 사용자가 결과를 확인하고 피드백을 한다.
      1. 피드백간 사용자가 해당 내용을 확인 할 수 있도록 방법은 제시한다.
         1. 해당 프로젝트의 로컬 서버를 띄워 보여주거나, 테스트 방법을 제시하거나 해야한다.
   2. 피드백할 사항이 없으면 완료를 눌러 해당 테스크를 완료처리한다.
5. 진행간 완료에 도달할 만한 사항이 아니면 backlog로 보내 기록을 남기고, 변경사항은 롤백한다.

## 사용자 요구사항 vs 현재 스펙 Gap 분석

|---------|---------|-----|

| 2. 프로젝트별 태스크 필터링 | 없음 | Phase 1 Task에 project_id 필드 추가 |

| 5. 자동 보안/코드 리뷰 | 없음 | Phase 5에 Quality Pipeline 추가 |
| 6. 개발자 승인 UI | Phase 4에 간단히만 | Phase 4에 Review Workflow UI 추가 |
